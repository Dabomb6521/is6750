ProductContext.  Presently, the CategoryMenu, CategoryList, FeaturedProducts, ProductsByCategory, and ProductDetail components all retrieve data from the DummyJSON API each time the component is loaded.  This means that (a) there are multiple API calls each time a link is clicked (e.g., the CategoryMenu reloads every time!) and (b) each component needs to have its own data fetching functions.  To address these issues, do the following:

  1. When the component first loads, it should retrieve an array of all products from the browser’s sessionStorage (sessionStorage works just like localStorage, except the data only persists while the user’s browser tab is open) and save it in a state variable.  If the data does not exist in sessionStorage, the component should retrieve the data from the API using the /products?limit=0 endpoint, and then save the data in a state variable and in sessionStorage for subsequent requests.  (Hint:  sessionStorage stores data in string format, so when adding or retrieving array or object data, use JSON.stringify() and JSON.parse(), respectively.

  4. getProduct(productID): Retrieves a specific product by the given ID
  1. The component should return a ProductContext.Provider component with pointers to these functions in its value prop.
